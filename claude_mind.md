## 二、核心问题定义

### 研究目标
**参数→决策端到端**：给定发动机传感器时序数据（172维特征），直接输出最优维修决策（继续运行/预防性维修/事后维修/更换），无需人工中间环节。

**跨工况跨故障自适应**：同一策略在面对不同工况（高度/马赫数/油门角）、不同故障模式（HPT效率退化/LPT流量退化/双重故障）的发动机时，能以少量历史交互快速适配，无需重新训练。

### 问题形式化
```
MDP定义:
  状态 s_t = [h_t; pos_t]  ∈ R^65
    h_t  = GRU(x_{1:t}) ∈ R^64   # 健康状态嵌入
    pos_t = t / T ∈ [0,1]         # 退化进度
  
  动作 a_t ∈ {0,1,...,7}          # 8种维修决策
  
  奖励 r_t = 运营收益 - 维修成本 - 故障惩罚
  
  任务分布 p(M):
    M = (工况配置, 故障模式, 发动机个体)
    训练任务: unit 16, 18, 20（历史见过）
    测试任务: unit 14, 15（零样本/少样本迁移）

元学习目标:
  θ* = argmax E_{M~p(M)} [E_{τ~π_{θ,z}} [R(τ)]]
  其中 z = f_φ(context) 为任务隐变量
```

## 三、系统架构

```
┌──────────────────────────────────────────────────────────────┐
│                    传感器原始数据流                            │
│  N-CMAPSS: Alt, Mach, TRA, T2 + 30传感器 → (T × 43 × 4)维  │
└─────────────────────────┬────────────────────────────────────┘
                          │ 时频特征提取 + 质量增强
                          ▼
┌──────────────────────────────────────────────────────────────┐
│              退化状态编码器（GRU Encoder）                    │
│   输入: (T, 172)   →   GRU(2层, hidden=128)   →   h∈R^64   │
│   自监督预训练目标: 预测归一化退化进度 t/(T-1)               │
└─────────────────────────┬────────────────────────────────────┘
                          │ h_t + pos_t → s_t ∈ R^65
                          ▼
┌──────────────────────────────────────────────────────────────┐
│                   任务推断模块（Context Encoder）              │
│   context = {(s_i, a_i, r_i, s'_i)} × K                    │
│   → VAE: μ, σ → z ∈ R^8                                     │
│   z 编码当前发动机的工况+故障模式信息                         │
└──────────┬──────────────────────────┬────────────────────────┘
           │ z                         │ z
           ▼                           ▼
┌──────────────────┐       ┌─────────────────────────────────┐
│  Actor π(a|s,z)  │       │  Critic Q(s,a,z)×2 (Twin-Q)   │
│  MLP: (65+8)→8   │       │  MLP: (65+8)→8                  │
└──────────────────┘       └─────────────────────────────────┘
           │
           ▼ 维修决策
┌──────────────────────────────────────────────────────────────┐
│                  维修决策环境                                  │
│  8动作: 继续/预防A/预防B/预防A+B/事后A/事后B/事后A+B/更换    │
│  奖励: 运营收益(+10/步) - 维修成本 - 故障惩罚                │
└──────────────────────────────────────────────────────────────┘
```

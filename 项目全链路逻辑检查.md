# 项目全链路逻辑检查：数据预处理 → 决策

## 一、整体数据流

```
原始 H5 (N-CMAPSS)
    ↓ step2 / step3
CSV (unitXX_feature_all_freq.csv)
    ↓ step4.1 特征筛选
selected_features.pkl
    ↓ step4.2 质量增强 + 序列构建
sequences_complete.npy (滑动窗口) + trajectory_complete.npy (完整轨迹)
    ↓ step4.2-批量生成轨迹 (可选)
trajectory_complete.npy (unit 14,16,18,20)
    ↓ pretrain_gru.py (可选)
gru_pretrained.pt
    ↓ pearl.py
PEARL 训练 → Actor/Critic → 维护决策
```

## 二、各阶段逻辑

### 1. 数据预处理 (1数据处理/DS02)

| 步骤 | 文件 | 输入 | 输出 | 说明 |
|------|------|------|------|------|
| step2 | step2-预处理对应的列.py | H5 | combined_data.csv, N-CMAPSS_DS02_selected.csv | 选列、下采样、归一化 |
| step3.2 | step3.2加入频域特征提取.py | selected.csv | feature_all/unitXX/ | 频域特征、统计特征 |
| step4.1 | step4.1-特征筛选.py | XX_feature_all_freq.csv | selected_features.pkl | 互信息+相关性，保留 K 个传感器 |
| step4.2 | step4.2-质量增强与序列构建.py | selected_features.pkl + CSV | sequences_complete.npy, trajectory_complete.npy | 缺失/异常处理，序列构建 |

**特征维度**：`len(selected_features) × 4 = 172`（每特征：imputed, mask, delta, outlier）

### 2. 序列形式

| 类型 | 形状 | 含义 |
|------|------|------|
| sequences_complete.npy | (N, 30, 172) | 滑动窗口，N 个独立 30 步样本 |
| trajectory_complete.npy | (T, 172) | 完整退化轨迹，T=cycle 数 |

**load_sequences 优先级**：trajectory_complete.npy > sequences_complete.npy

### 3. GRU 预训练 (pretrain_gru.py)

- **任务**：输入 prefix `seq[:t+1]`，预测 `target = t/(T-1)`（归一化位置）
- **数据**：unit 14,16,18,20 的 trajectory
- **输出**：gru_pretrained.pt（GRU 权重，不含 regressor）

### 4. 环境 (env.py)

| 组件 | 逻辑 |
|------|------|
| **输入** | sequence (T, 172)，encoder (GRU) |
| **state** | GRU(sequence[:pointer+1]) + pointer/seq_len → (65,) |
| **pointer** | 0=健康，seq_len-1=退化 |
| **动作** | 0 运行 +15，1-3 提前维修 +7/+5，4-6 事后维修 负，7 更换 -20 |
| **维修** | 第 n 次维修回退到 (n-1)*step_size，连续 15 次早停 |

### 5. PEARL 决策 (pearl.py)

| 组件 | 逻辑 |
|------|------|
| **训练数据** | unit 16, 18, 20 的 trajectory |
| **测试数据** | unit 14 的 trajectory |
| **Context** | 每 episode 重置，从当前 episode 的 (s,a,r,s2) 推断 z |
| **Actor** | (state, z) → 动作概率 |
| **Replay** | 仅 episode_reward > -1500 时加入 |
| **Curriculum** | 前 100 ep 压制动作 4/5/6 |

## 三、潜在问题与一致性

### 1. 数据路径依赖

- step4.1 默认处理 **unit20**，保存到 `feature_all/unit20/feature_selected/`
- step4.2 批量脚本处理 unit 14,16,18,20，需各 unit 已有 `selected_features.pkl`
- **建议**：对 unit 14,16,18,20 分别运行 step4.1，或确认 step4.1 支持多 unit

### 2. 特征一致性

- step4.1 的 `selected_features` 需与 step4.2、pretrain、pearl 使用的特征一致
- 各 unit 的 `selected_features.pkl` 应来自同一特征筛选流程
- **当前**：step4.1 仅配置 unit20，其他 unit 可能需单独运行或共享 pkl

### 3. 轨迹长度

- trajectory 长度 T 随 unit 不同（如 63~76）
- env 的 MAX_LEN=200，实际受 `min(200, len(seq))` 限制
- pointer 越界时 done，reward -= 20

### 4. 动作 A/B 与数据

- 动作 1-6 对应「部件 A/B」为抽象概念，数据无显式 A/B 标签
- 策略在 8 个动作间学习，A/B 为建模假设

### 5. 预训练与决策

- 预训练目标：位置回归 `t/(T-1)`
- 决策 state 含 GRU(64) + position(1)，position 与预训练目标一致
- GRU 编码序列模式，position 提供显式位置信息
